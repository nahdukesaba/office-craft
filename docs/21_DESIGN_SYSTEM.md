# Design System

## Principle

Office Craft already has an established visual language.

Future development must preserve it.

Consistency is more valuable than novelty.

---

# Typography

Maintain existing hierarchy.

Do not introduce additional heading scales.

Avoid mixing font styles.

---

# Spacing

Reuse existing spacing rhythm.

Avoid arbitrary padding.

Avoid inconsistent margins.

---

# Radius

Respect existing border radius.

Avoid introducing another radius system.

---

# Colors

Use semantic colors.

Primary

Secondary

Success

Warning

Danger

Muted

Avoid one-off colors.

---

# Shadows

Use existing elevation scale.

Do not invent new shadow patterns.

---

# Icons

Lucide is the canonical icon library.

Avoid introducing another icon library.

---

# Tables

Tables should remain visually identical across modules.

Features should customize columns,

not table implementations.

---

# Forms

Every form should feel familiar.

Input

↓

Validation

↓

Error

↓

Submit

↓

Feedback

Maintain this flow.

---

# Dialogs

Dialogs solve focused tasks.

Complex workflows deserve dedicated pages.

---

# Empty States

Every feature should implement:

Loading

Empty

Error

Success

Avoid blank screens.

---

# Notifications

Use one notification style across the application.

Avoid feature-specific notification systems.

---

# Responsive Design

Responsive behavior should extend existing layouts.

Avoid creating separate mobile pages.

---

# AI Rule

Visual changes require explicit instruction.

Silently redesigning UI is considered a regression.
