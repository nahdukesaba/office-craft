# Implementation Patterns

## Adding a CRUD Module

1.

Create backend resource.

2.

Expose endpoints.

3.

Create frontend API client.

4.

Create query hooks.

5.

Create mutations.

6.

Reuse DataTable.

7.

Reuse ConfirmDialog.

8.

Reuse Form components.

9.

Invalidate queries.

10.

Update navigation.

---

## Adding a Dashboard Widget

1.

Backend aggregation.

2.

Typed endpoint.

3.

Query hook.

4.

Card component.

5.

Loading state.

6.

Empty state.

---

## Adding File Upload

1.

Backend endpoint.

2.

Frontend upload component.

3.

Progress.

4.

Refresh.

---

## Adding Permission

Backend first.

Frontend second.

Never frontend first.

---

## Refactoring

Before refactoring:

Search imports.

Search reuse.

Search hooks.

Search stores.

Search types.

Search utilities.

Only refactor if duplication genuinely exists.
