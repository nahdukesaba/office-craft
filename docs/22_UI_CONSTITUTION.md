# UI Constitution

The following rules are mandatory.

Violation requires explicit user approval.

---

Rule 1

Never redesign an existing page.

---

Rule 2

Never regenerate an existing layout.

---

Rule 3

Never replace working components.

---

Rule 4

Never rename reusable components without instruction.

---

Rule 5

Never move files solely for organizational reasons.

---

Rule 6

Never replace TanStack Router.

---

Rule 7

Never replace Zustand.

---

Rule 8

Never replace TanStack Query.

---

Rule 9

Never replace Tailwind.

---

Rule 10

Never replace Radix UI primitives.

---

Rule 11

Never introduce another design language.

---

Rule 12

Every feature should reuse:

Cards

Tables

Dialogs

Buttons

Inputs

Layouts

before introducing new ones.

---

Rule 13

Every new page should resemble existing pages.

---

Rule 14

Visual consistency has higher priority than visual improvement.

---

Rule 15

If unsure,

ask before redesigning.

---

Rule 16

The repository is always the source of truth.

Not the AI's preferences.

---

Rule 17

Modify the smallest responsible component.

Never rewrite an entire page to change one behavior.

---

Rule 18

Read AI_CONTEXT.md before writing code.

Always.
